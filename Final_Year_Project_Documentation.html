SYSTEM ARCHITECTURE AND IMPLEMENTATION

1. Introduction

This project, SmartGuard IPS (Support Vector Machine Intrusion Prevention System), is designed to classify network traffic into Normal or Malicious categories using Machine Learning. The system employs a modern 3-tier architecture separating the client interface, the machine learning server logic, and the persistent database storage. This ensures scalability, security, and real-time performance.

2. How the Components Work Together

The system is built on three major pillars which communicate seamlessly to provide real-time intrusion detection and prevention:

A. The Frontend (Client-Side)

Hosted on Firebase Hosting, the frontend is built with HTML5, CSS3, and Vanilla JavaScript. It serves as the user interface where network administrators interact with the system. It collects user inputs (such as uploading PCAP or CSV network files) and displays visual analytics using Chart.js. The frontend communicates with the backend via RESTful API calls using the Fetch API.

B. The Python Backend (Server-Side & Machine Learning)

Hosted on Vercel as Serverless Functions, the backend is built using FastAPI and Python. This is the brain of the system. It houses the pre-trained Support Vector Machine (SVM) model (built with Scikit-learn). When the frontend sends network data, the FastAPI server extracts the numerical features, feeds them into the SVM model, and immediately returns the prediction (Malicious/Blocked or Normal/Allowed) back to the user without freezing the user's browser.

C. The Database & Authentication (Supabase)

The system uses Supabase (a PostgreSQL cloud database) to handle two main things: User Authentication and Data Persistence. When a user creates an account, Supabase handles the security and issues a secure session token. When the Python backend detects network threats, the administrator can save these logs directly into the PostgreSQL `traffic_logs` table. The frontend later fetches these saved logs to generate historical reports and dashboard statistics.

4. Module by Module UI and Functionality Explanation

Module 1: Authentication Module

This module manages user access control. It prevents unauthorized individuals from accessing sensitive network logs. It integrates directly with Supabase Auth to handle user registration, secure login, and session storage.

User Interface: Features a sleek, dark-themed login and registration form asking for Email, Password, First Name, and Last Name. It includes validation messages and a secure password masking toggle.
Example Scenario: A new security admin enters their email admin@smartguard.com and a secure password. The UI displays a loading spinner, communicates with Supabase, and redirects the user to the dashboard upon successful authentication.

Module 2: Upload and Detection Module

This is the core functional module. It handles file parsing, data preprocessing, and SVM threat prediction by sending data to the Python backend.

User Interface: Features a prominent drag-and-drop zone for .csv or .pcap files. Once a file is dropped, an animated progress pipeline appears (Parsing → Feature Extraction → SVM Classification). Below this, a dynamic tabular results grid renders the Source IP, Destination IP, Protocol, Duration, and Prediction.
Example Scenario: The admin drops a traffic_capture.pcap file. The UI transitions through the progress steps smoothly. The table then dynamically populates with thousands of records, highlighting malicious threats with a stark red "Blocked" badge, and benign traffic with a green "Allowed" badge.

Module 3: Live Monitoring Module

This module simulates a real-time network environment, polling the backend for incoming traffic streams.

User Interface: Designed to mimic a live terminal feed. It contains a scrolling list of incoming network packets. The UI includes pulsing status dots and smooth slide-in animations for each new packet to indicate active, real-time monitoring.
Example Scenario: The UI polls the API every few seconds, dynamically appending new packets to the top of the feed. The admin watches as a 'Normal' packet passes through, followed immediately by a 'DoS Attack' which visually triggers a red alert notification in the feed.

Module 4: Dashboard and Statistics Module

Data visualization is crucial for summarizing network health. This module fetches aggregated historical data from the database.

User Interface: A comprehensive analytics screen featuring interactive Chart.js visualizations. It contains a donut chart for traffic distribution (Normal vs. Malicious), a bar chart detailing specific attack signatures, and high-level metric cards (e.g., "Total Records Processed", "Threats Detected").
Example Scenario: The admin logs in and immediately sees a dashboard summary indicating that 150 threats were blocked today. The interactive donut chart visually emphasizes that 5% of all incoming traffic was malicious.

Module 5: Reports and Evaluation Module

This module serves compliance auditing and academic research purposes, displaying both raw logs and Machine Learning performance metrics.

User Interface (Reports): A robust historical log table with advanced dropdown filters (Filter by Action, Filter by Type) and a button to "Export CSV".
User Interface (Evaluation): Displays ML metrics in an academic layout, using animated circular progress bars to indicate Model Accuracy, Precision, Recall, and F1-Score.
Example Scenario: The admin filters the reports table for 'Blocked' actions only, reviews the targeted intrusive IP addresses, and clicks 'Export CSV' to download a spreadsheet for their monthly security audit report.

5. Machine Learning Model Training

The core intelligence of the system relies on a mathematical model that was trained in Python. The process is handled by a dedicated training script (train_dummy_model.py).

  • Data Generation: The script utilizes the scikit-learn library (specifically make_classification) to generate synthetic dataset features. It creates 100 sample network packets, each defined by 10 specific numerical features (such as duration, byte size, etc.).
  • Algorithm Selection: The system employs a Support Vector Machine (SVM) classifier. Specifically, an SVC (Support Vector Classifier) with a linear kernel is chosen. SVMs are highly effective in cybersecurity because they can plot multi-dimensional data and find the optimal "hyperplane" that cleanly separates benign traffic from malicious traffic.
  • Training the Model: The synthetic data (features X and labels y) is fed into the algorithm using the model.fit(X, y) function. This process allows the algorithm to learn the underlying patterns that signify a cyber attack.
  • Model Serialization: Once the model has successfully learned the patterns, it is serialized (saved) to a binary file named svm_model.pkl using the joblib library. This file is stored in the api/model/ directory.
  • Real-Time Inference: By saving the model to a .pkl file, the FastAPI backend does not need to retrain the model every time a user uploads a file. Instead, the backend simply loads the pre-trained model into memory and executes the predict() function instantly on new incoming data.
