🗂️ Folder Structure (Vercel-Compatible)
Vercel looks for an api/ folder at the root of your project. Your project will be structured like this:

svm_intrusion_prevention_system/
├── api/                        ← Vercel runs everything from here
│   ├── index.py                ← FastAPI entry point (main.py renamed)
│   ├── upload.py               ← /api/upload endpoint
│   ├── live_traffic.py         ← /api/live-traffic endpoint
│   ├── stats.py                ← /api/stats endpoint
│   └── model/
│       └── svm_model.pkl       ← Your trained SVM model file
├── requirements.txt            ← Python dependencies for Vercel
├── vercel.json                 ← Vercel config file
└── public/                     ← Your Firebase frontend (already done ✅)
⚙️ Step 1: FastAPI Setup
1.1 — Create the api/ folder at the root (not inside public).

1.2 — Create requirements.txt at the root. Vercel reads this to install your Python packages:

fastapi
scikit-learn
pandas
python-multipart
supabase
joblib
1.3 — Create vercel.json at the root. This tells Vercel how to run your Python app:

json
{
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/index.py" }
  ]
}
1.4 — Create api/index.py — this is your FastAPI app with CORS:

python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
app = FastAPI()
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://cyber-shield-svm.web.app"],  # Your Firebase URL
    allow_methods=["*"],
    allow_headers=["*"],
)
Local testing only: Install uvicorn locally (pip install uvicorn) and run uvicorn api.index:app --reload. Vercel doesn't need this in production.

🤖 Step 2: Machine Learning Integration
2.1 — Train & Export your SVM model (if not done already):

python
import joblib
from sklearn.svm import SVC
# ... train your model ...
joblib.dump(model, "api/model/svm_model.pkl")
2.2 — Load the model in api/index.py once on startup:

python
import joblib, os
MODEL_PATH = os.path.join(os.path.dirname(__file__), "model", "svm_model.pkl")
model = joblib.load(MODEL_PATH)
2.3 — Build /api/upload in api/upload.py:

Accept a .csv file via POST
Read with pandas
Select the right feature columns your SVM was trained on
Run model.predict(features)
Return JSON: { "results": [...] }
2.4 — Build /api/live-traffic in api/live_traffic.py:

Use FastAPI's StreamingResponse to send batches of mock/real packet data
Each packet gets model.predict() run on it
Returns a JSON stream the frontend polls every few seconds
⚠️ Vercel limitation: WebSockets are not supported on Vercel. Use regular HTTP polling (setInterval on the frontend calling /api/live-traffic every 2–3 seconds) instead.

🗄️ Step 3: Supabase Integration
3.1 — Create a Supabase project at supabase.com → Get your SUPABASE_URL and SUPABASE_KEY.

3.2 — Create the table in Supabase SQL Editor:

sql
CREATE TABLE traffic_logs (
  id         BIGSERIAL PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT now(),
  source_ip  TEXT,
  dest_ip    TEXT,
  protocol   TEXT,
  prediction TEXT,   -- 'Normal' or 'Malicious'
  action     TEXT    -- 'Allowed' or 'Blocked'
);
3.3 — Set Environment Variables on Vercel (never hardcode keys):

Go to Vercel Dashboard → Your Project → Settings → Environment Variables
Add SUPABASE_URL and SUPABASE_KEY
3.4 — Connect Python to Supabase in api/index.py:

python
import os
from supabase import create_client
supabase = create_client(os.environ["SUPABASE_URL"], os.environ["SUPABASE_KEY"])
📊 Step 4: Analytics Endpoint
4.1 — Build /api/stats in api/stats.py:

Query traffic_logs from Supabase
Count total packets, malicious packets, attack type breakdown
Return structured JSON ready for Chart.js:

🚀 Deployment to Vercel
Once everything is coded:

bash
# 1. Install Vercel CLI
npm install -g vercel
# 2. Login
vercel login
# 3. Deploy from project root
vercel --prod
Vercel will auto-detect your api/ folder and deploy all Python functions as serverless endpoints at https://your-project.vercel.app/api/...
